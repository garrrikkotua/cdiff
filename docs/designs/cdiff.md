# cdiff — Browser Diff Viewer for AI Coding

Generated by /plan-ceo-review on 2026-03-20
Branch: main | Mode: SCOPE EXPANSION

## Problem

When using AI coding agents (Claude Code, Codex) in the terminal, you make changes across many files but have no lightweight way to see the full picture. `git diff` is hard to scan. Opening VS Code/Cursor just to glance at changes is a context-switch tax. There's no tool purpose-built for this.

## Vision

cdiff is the **missing feedback loop for AI-assisted coding**. You invoke Claude, changes start happening, you open cdiff, and instead of anxiety ("what did it change?") you feel clarity ("ah, I see exactly what it did").

### 10x Vision
The command-center for AI coding sessions — a live-updating dashboard showing every file touched, with syntax-highlighted diffs, file tree navigation, and auto-refresh as changes land.

## Accepted Scope (v1)

### Core
- Local HTTP dev server with Bun.serve
- VS Code-style file tree sidebar with change indicators (M/A/D/R)
- Unified diff view with line numbers
- File watching + SSE for live auto-refresh (300ms debounce)

### Expansions (all accepted)
1. **Staged vs Unstaged toggle** — two views of git state
2. **Dark/Light theme** — OS detection via `prefers-color-scheme` + manual toggle
3. **Branch comparison** — `cdiff main..HEAD` for PR review
4. **Side-by-side diff view** — toggle between unified and split
5. **Syntax highlighting** — language-aware coloring via highlight.js or Prism
6. **Dual distribution** — standalone CLI (`npm i -g cdiff`) + Claude Code skill (`/cdiff`)

## Tech Stack
- **Runtime:** Bun (compiles to single binary via `bun build --compile`)
- **Server:** Bun.serve (HTTP + SSE)
- **Frontend:** Vanilla HTML/CSS/JS (no framework, no build step for UI)
- **Syntax highlighting:** highlight.js (bundled into binary, offline-first)
- **Distribution:** npm, GitHub Releases (binary), Claude Code skill

## Engineering Decisions (from /plan-eng-review)

### Diff Mode State Model
```
cdiff                → working directory mode (default)
                       Shows All/Staged/Unstaged toggle + file watcher active
cdiff main..HEAD     → branch comparison mode
                       Staged/unstaged toggle HIDDEN, file watcher DISABLED
```
Modes are mutually exclusive based on CLI args.

### Lazy Loading
Initial page load fetches only `git diff --name-status` (file tree).
Individual file diffs loaded on click via `/api/diff?file=<path>`.
SSE refresh re-fetches tree only; currently-viewed file re-fetched if changed.

### Syntax Highlighting
highlight.js bundled into the compiled binary (offline-first). No CDN dependency.

### Merge Conflicts
When `.git/MERGE_HEAD` exists, show diffs as-is with conflict markers rendered,
plus a yellow warning banner: "Merge in progress — conflict markers shown."

### Git Diff Parser Edge Cases
Binary files → placeholder. Renames → R indicator with old→new path.
Mode-only changes → mode change indicator. Submodules → commit hash change.
No-newline-at-EOF → dim annotation. Huge files (>5K lines) → truncated.
Empty files → "(empty file)" placeholder. Unicode filenames → handle quoted paths.
